# CHECKLIST-UX

- Описано ли поведение, если нет ни одной главы/темы/мема/проекта? UI по пустому состоянию, подсказка.
- UX сценарии: есть ли инструкции при ошибке/зависании генерации? Как пользователь понимает, что делать дальше?
- Ошибки генерации/конвертации/загрузки — снабжены понятными пояснениями для пользователя?
- UX-поток загрузки и прогресса генерации ясны? Отдельные индикаторы для каждой главы/ресурса?
- Продумана ли логика ручного выбора музыки, SFX? Есть fallback и возможность отказаться?
- Валидация лимитов загрузок медиа (размер, допустимый формат). UX ошибки — с объяснением.
- Состояния неудачи (нет AI-ключа, ошибка сети, не удалось создать видео) хорошо отражены модально/уведомлением?
- Гость/авторизация — все основные функции доступны без логина, защищена приватность?
- Логика переключения между проектами, возможности повторной генерации, история понятны?
