# Clarify: уточнение бизнес- и технических требований

- Минимальный размер входящего текста/темы: 2 слова. Поддерживается ручное описание или загрузка "базы знаний".
- Авторизация не требуется, весь прогресс — локально (LocalStorage, IndexedDB).
- Максимальный размер аудиофайла в проекте: 20 МБ; изображений — 5 МБ/штука; готовое видео — 200 МБ.
- Видео собирается напрямую в браузере, скорость зависит от железа и наличия SharedArrayBuffer.
- Неограниченное количество глав, но для оптимальной UX — рекомендовано от 2 до 8 глав.
- Голоса TTS: fallback на OpenRouter при ошибке Gemini TTS.
- SFX-эффекты — только через Freesound, с автопереводом запросов.
- Генерация YouTube-обложек: только из сгенерированных/загруженных картинок, не из интернета.
- Все ошибки фатальных AI-сервисов — уведомление пользователя и предложение сменить ключ/platformу.
- Никакие пользовательские данные не уходят наружу (privacy by design).
- Нет серверной логики, только статика, никакой регистрации и PII.
- API-ограничения: 10k знаков в сценарии на одну главу, 200 секунд итогового видео для MVP.
